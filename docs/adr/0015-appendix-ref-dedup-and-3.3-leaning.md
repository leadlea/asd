# ADR-0015: 「付録付録A」重複の解消と3.3の結合キー簡潔化

- 状態: Accepted
- 日付: 2026-06-22
- 決定者: 福原玄（NCNP共同研究・筆頭）
- 関連: `paper1_ja.tex`（3.3 ほか）, ADR-0004/0006/0014

## コンテキスト

1. PDFで「付録付録A 参照」と**付録が二重**に出力されていた。
2. 3.3（コーパス基本情報との関連性）に「具体的なカラムのキー名（conversation_id × cejc_person_id）」を入れるべきか、という問い。

## 原因（付録二重）

本文書は `\appendix` 配下で `\ref` が自動的に「付録A」を返す設計（既存箇所はすべて素の `\ref{sec:appendix_*}` を使用）。ADR-0004/0012/0014で補足参照を追加した際に「付録\ref{...}」と接頭辞を手で付けたため、「付録」＋「付録A」＝「付録付録A」となっていた。

## 決定

1. 付録参照は本文書の慣例どおり**素の `\ref{}`** に統一（接頭辞「付録」を除去）。該当5箇所（sec:supp_dataset×2, sec:appendix_cv, sec:supp_corr, コメント1）を修正。
2. カラムキーの是非（回答）: **Results(3.3)には入れない**。結合キー（conversation_id × cejc_person_id）やカラム名は実装詳細で、既に付録A（補足）に保全済み。Resultsは結果に集中させ可読性を優先する。2.1での同種削除（ADR-0006）とも整合。
   - 3.3の「conversation_id × cejc_person_idをキーに紐付けた」「age列」を削除し、「CEJCメタ情報を付録Aの手順で紐付けた」に簡潔化。

## 検証

- `付録\ref{` 残存: 0（本文・コメント）。
- uplatex 2パス＋dvipdfmx: 正常、47ページ、未解決の参照・引用なし。

## 申し送り

- もし結合キー/カラム名を明示したい場合は、Resultsではなく付録A（sec:supp_dataset）に追記するのが妥当（要指示）。
